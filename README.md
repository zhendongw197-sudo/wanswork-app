# 营销工单助手

一款面向供电营销低压业扩工单管理的 Windows 单机工具（tkinter 界面），集工单台账、时限预警、配套材料一键生成、Excel 导出于一体。

## 功能简介

- **工单台账**：六种低压流程（低压居民新装 / 低压居民增容 / 低压充电桩新装 / 低压充电桩增容 / 低压非居民新装 / 低压非居民增容）的新增、修改、删除、进度编辑。
- **时限引擎**：按“工作日 24 小时累计制”自动计算上门服务截止日与全流程截止日；自动跳过周末、节假日，支持调休上班日；起算时间自动对齐（周末/节假日顺延、早于 8:30 按 8:30、17:30 后顺延下一工作日 8:30）。时限：低压居民新装/增容全流程 5 个工作日（无上门服务截止日）；低压充电桩新装/增容全流程 15 个工作日 + 上门服务 3 个工作日；低压非居民新装/增容全流程 15 个工作日 + 上门服务 4 个工作日。
- **临期预警**：主表格按紧急程度排序，超期标红、临期标黄，每 60 秒自动刷新。
- **终止材料生成**：一键生成仿 iPhone 通话记录截图、仿 iMessage 短信确认截图、终止白名单情况说明（Word）。
- **四方交底材料生成**：一键生成交底申请书、施工方案、监理情况说明（3 份 Word）。
- **供方单生成与台账**：按模板生成供电方案确认单（Word），自动推送至供方单台账；台账支持编辑进度、删除、导出 Excel。
- **导出与汇总**：工单列表导出 Excel（带表头样式、冻结首行）；按流程类型 / 状态汇总统计。

## exe 使用方法

1. 双击 `营销工单助手.exe` 即可运行，无需安装 Python。
2. 首次运行会在 **exe 所在目录** 自动创建两个文件夹：
   - `data\`：存放全部业务数据（`store.json`，含工单、客户经理、节假日、模板配置、供方单台账）。**请勿删除**，备份数据只需复制该文件夹。
   - `output\`：存放生成的材料，按“终止材料_流程编号”“四方交底_流程编号”“供方单_流程编号”分目录存放。
3. exe 所在目录还会出现一个 `paths.json`：记录当前生效的**数据路径**与**输出路径**（在【设置】→“路径配置”页签修改后自动写入）。删除该文件则恢复默认（exe 目录下的 `data\`、`output\`）。
4. 建议把 exe 放在一个固定目录（如 `D:\营销工单助手\`）中使用；移动 exe 时把 `data\` 一起搬走即可保留数据（若已改到其他数据路径，则搬迁对应目录即可）。

## 五类功能操作步骤

### 1. 工单录入与管理
- 点【新增工单】，填写流程编号、流程类型、户号、户名、客户经理、客户手机号、地址、流程开始日期（格式 `YYYY-MM-DD HH:MM:SS`），确定保存。地址为多行文本框，支持换行输入较长地址。
- 选中表格中一行，可点【修改】（或双击该行）、【删除】、【进度编辑】。
- 【进度编辑】弹窗显示当前进度，四选一：**已建群**、**上门服务办结**、**流程办结**、**终止**（“上门服务办结”仅低压充电桩/低压非居民类工单可选，低压居民类不显示）。选“已建群 / 上门服务办结 / 流程办结”时“是否建群”自动置“是”；选“终止”时保留原值。
- 考核规则：流程办结 / 终止后不再考核任何截止日，剩余时间显示 `-`、不标色、排序沉底；上门服务办结后只考核全流程截止日（上门服务截止日不再参与颜色与剩余时间计算）；在办 / 已建群维持双截止日考核。

### 2. 生成终止材料
- 选中工单 →【生成终止材料】→ 核对/补全客户手机号、地址，填写终止原因、通话时长、电量 →【生成】。
- 输出到 `output\终止材料_流程编号\`：材料1-通话记录截图.png、材料2-短信确认截图.png、材料4-终止白名单情况说明.docx。手机号与地址会自动回写到工单。

### 3. 生成四方交底材料
- 选中工单 →【生成四方交底材料】→ 填写手机号、地址、街道、开挖距离、交底日期（格式 `2026年7月10日`），选择客户经理 →【生成】。
- 输出到 `output\四方交底_流程编号\`：交底申请书.docx、施工方案.docx、监理情况说明.docx。

### 4. 生成供方单与台账
- 选中工单 →【生成供方单】→ 在弹窗中核对/填写基本信息（供电电压、批准容量、线路、变压器名称、合同容量、去年负荷、接户方式、电表电流型号、电价、行业分类、客户手机号、地址）、表箱（单相/三相、表位、悬挂式/落地式）与材料明细（17 项，无则留空）→【确定】。其中行业分类为客户经理手填（初值按流程类型预填，可自由修改）。
- 输出到 `output\供方单_流程编号\供电方案确认单.docx`；生成后自动写入供方单台账（推送时间为当前时间，超期时间取全流程截止日），手机号与地址自动回写到工单。若模板中有未匹配的红字占位，会弹窗列出占位名。
- 点【供方单台账】打开“已推送供方单”窗口：【刷新】重载列表；【编辑进度】（或双击行）编辑是否归档、当前进度（营销部反馈）、设备部进度（是否发料）、备注、配套单位 5 个手工字段；【删除】需二次确认；【导出Excel】导出全部台账记录。

### 5. 导出 Excel 与汇总
- 【导出Excel】：选择保存路径，导出当前全部工单（含实时计算的截止日与剩余时间）。
- 【汇总】：弹窗显示工单总数、按流程类型统计、按状态统计。

## 节假日 / 调休维护

- 点【设置】→“节假日与调休”页签。
- 左侧文本框填**节假日**（每行一个 `YYYY-MM-DD`，如 `2026-10-01`）；右侧填**调休上班日**（周末上班的日期）。
- 点【保存】后主表格立即按新日历重算全部截止日与标色。
- 规则：周一至周五且不在节假日中为工作日；调休上班日无论周几都按工作日算；调休与节假日同一天时按上班算。

## 模板配置说明

- 点【设置】→“模板配置”页签，共 7 项，用于生成 Word 材料时填充：

| 配置项 | 用途 |
| --- | --- |
| 燃气公司 | 交底申请书抬头 |
| 施工单位 | 施工方案落款 |
| 施工负责人 | 施工方案落款 |
| 施工电话 | 施工方案落款 |
| 落款单位 | 申请书/施工方案的建设单位全称 |
| 公司简称 | 短信截图中供电公司简称 |
| 开挖深度 | 施工方案中的开挖深度（厘米，填数字） |

- 修改后点【保存】即生效，随数据持久化在 `data\store.json`。

## 路径配置说明

- 点【设置】→“路径配置”页签，可修改两个路径：
  - **数据路径**：`store.json` 所在目录，即全部业务数据的存放位置；
  - **输出路径**：终止材料 / 四方交底 / 供方单等生成材料的保存根目录。
- 可手输或点【浏览…】选择；**支持内网服务器共享路径**（如 `\\服务器\共享\工单数据`），多台电脑指向同一共享目录即可共用数据；相对路径基于软件所在目录（exe 目录）解析。
- 点【保存】后**即时生效、无需重启**：目录不存在时会询问是否自动创建；配置写入 exe 所在目录的 `paths.json`，下次启动继续生效。
- **数据迁移提示**：修改数据路径后，若新目录下没有 `store.json`，会询问“是否把现有数据复制到新路径”，选“是”则自动复制，原目录数据保留不删。

## 源码运行方式

要求 Python 3.12+（依赖：tkinter 自带、openpyxl、python-docx、pillow）：

```bat
pip install openpyxl python-docx pillow
python main.py
```

源码运行时 `data\`、`output\` 创建在项目根目录下。

## 重新打包方法

双击项目根的 `build_exe.bat` 即可自动完成：安装 PyInstaller → 单文件、无控制台窗口打包 → 产物在 `dist\营销工单助手.exe`。

手工打包等价命令（项目根目录下）：

```bat
python -m pip install pyinstaller
python -m PyInstaller --noconfirm --clean --onefile --windowed --name 营销工单助手 main.py
```

打包后把 `dist\营销工单助手.exe` 拷到目标机器直接使用即可（Windows 64 位）。

## 已知限制

- 仅支持 Windows（中文字体取 `C:\Windows\Fonts\` 下的微软雅黑/宋体）。
- 单机单用户，无多机协同与权限管理；数据为明文 JSON，请妥善保管 `data\`。
- 节假日需每年手工维护，不自动同步国务院放假安排。
- exe 首次启动需解压内置运行时，约 2~5 秒，属正常现象。
- 截图中的时间、电量、通话时长为生成时刻/手填值，非真实手机数据，仅供业务示意。
